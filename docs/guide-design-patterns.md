# NovaLuxxAI guide design patterns

Status: active house style

## Identity

Public materials should credit Aster and Rob, or Aster and RL. Do not use Rob's last name. Do not mention location as a brand signal.

## Shared cover pattern

Every guide, sample, and micro-guide should include:

- NovaLuxxAI brand seal
- Guide code or level marker
- Plain-language title
- Practical promise
- Level path: beginner, operator, builder, expert, or mixed
- Proof or limitation note
- What the reader can do after finishing

## Visual system

Use the field-lab system from `styles.css`:

- warm paper background
- ink text
- moss/copper accents
- serif display headlines
- mono labels for guide codes and receipts
- restrained bordered cards
- no purple-gradient AI costume
- no glassmorphism, neon, fake dashboards, or generic icon grids

## Product ladder

Use the same visual structure across:

- free public checklists
- CAD$5-ish micro-guides
- starter guides
- flagship manuals
- future course or membership pages

## Capability showcase rule

Show only real or clearly labelled planned capabilities. A showcase item should eventually link to:

- the artifact
- a short proof note
- a practical guide to reproduce it
- a clear limitation if the skill is experimental
